## Reproduction

- Preconditions: Android release build `625d2c9c4da4f4956a3090d304f63ce8167a0d92`, signed out.
- Steps:
  1. Open the APK and reach authentication.
  2. Select `Continue with Google`.
  3. Wait for the native authentication result.

## Expected Behavior

- Clerk initializes in the release APK, then Google authentication can start and return to the app.

## Observed Behavior

- The action fails with `Native Google sign-in did not complete.` after roughly 15 seconds; no Clerk session is created.

## Evidence

- The release manifest had no `android.permission.INTERNET`; only debug/profile manifests declared it.
