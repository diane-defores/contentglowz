# Bug Record: BUG-2026-05-05-001

## Summary

- Title: Android system nav overlaps demo onboarding
- Current status: `closed`
- Severity: `medium`
- Owner: unassigned
- Next command: `none`

## Reproduction

- Preconditions:
  - Android app build installed on a device or emulator with the native system navigation bar visible.
  - User is on the entry screen.
- Steps:
  1. Tap `Open Interactive Demo`.
  2. Inspect the bottom of the demo onboarding pages.
  3. Compare with the authenticated dashboard shell bottom navigation.

## Expected Behavior

- Demo onboarding content and bottom actions stay above the Android native system navigation bar.
- The behavior matches the authenticated dashboard bottom nav, which already respects the bottom safe area.

## Observed Behavior

- The Android native bottom navigation bar overlaps the bottom of the demo onboarding UI.
- The dashboard shell does not show the same issue.

## Evidence

- User report in chat on 2026-05-05.
- Relevant code before fix: `lib/presentation/screens/onboarding/onboarding_screen.dart` used a `Scaffold` body `Column` without a bottom `SafeArea`, while `lib/presentation/screens/app_shell.dart` wraps the mobile bottom nav in `SafeArea(top: false)`.
