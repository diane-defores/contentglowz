# Bug Record: BUG-2026-05-05-002

## Summary

- Title: Android back closes app during demo instead of navigating back
- Current status: `fixed-pending-verify`
- Severity: `medium`
- Owner: unassigned
- Next command: `/sf-test --retest BUG-2026-05-05-002 on Android device`

## Reproduction

- Preconditions:
  - Android app build installed on a device or emulator.
  - User opened the interactive demo and advanced past the first onboarding step.
- Steps:
  1. Tap `Open Interactive Demo`.
  2. Continue to the next demo onboarding step.
  3. Press the Android system back button.
  4. Press Android back again from a root dashboard tab.

## Expected Behavior

- In the middle of the demo onboarding wizard, Android back moves to the previous onboarding step.
- When Android back would truly close ContentFlow, the app asks for confirmation before exiting.

## Observed Behavior

- Android back can close the application without warning instead of moving back within the demo flow.

## Evidence

- User report in chat on 2026-05-05.
- Relevant code before fix: `OnboardingScreen` used a `PageView` controlled only by in-app buttons and did not intercept Android system back. `AppShell` did not confirm root-route back exits.
