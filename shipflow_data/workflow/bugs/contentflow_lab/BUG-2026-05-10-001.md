# Bug Record: BUG-2026-05-10-001

## Summary

- Title: Personas draft SQL insert fails on Hrana None parse
- Current status: `needs-repro` (auth non valide en retest production-like)
- Severity: `medium`
- Owner: Diane
- Next command: `/sf-test --retest BUG-2026-05-10-001`

## Reproduction

- Preconditions:
  - Utilisateur connecté avec appel à `/api/personas/draft`.
  - Le backend utilise Turso/libsql (Hrana) avec `TURSO_*` configurés.
- Steps:
  1. POST `/api/personas/draft` avec un cas prefill IA (`project_repo`/`suggest_from_repo`).
  2. Suivre immédiatement le job via `/api/personas/draft-jobs/<id>`.
  3. Observer l’erreur SQL de parsing dans les logs backend.

## Expected Behavior

- La création du job IA ne doit pas échouer sur `None` provenant d’un paramètre nullable.
- Le job doit passer en `running` puis en `completed` ou `failed` avec message métier.

## Observed Behavior

- Erreur `Hrana: SQL string could not be parsed: near VALUES, "None": syntax error...` lors de l’insertion dans le store de jobs.
